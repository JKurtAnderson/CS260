Pass Off Drivers

Once you're done with your project, you should use these pages to run it against the TA pass off driver.

First 3 Labs

To pass off the first 3 labs, you will need to have a .zip archive and your netid. When you're ready, hover over the "Pass Offs" tab in the menu and select the lab that you're ready to submit. Once there, enter your netid and attach the .zip archive. It will then uncompress your code, run it, and report the results back to you.

Some Notes:

  • The .zip archive should contain only the code that you created/edited for this project. Do not include the interface files or any files from the pass off package.
  • You may not use any additional Jars for these projects. The pass off driver would have no way of grabbing these dependencies.
  • If you have any issues with the pass off driver, check out the FAQ page.

Famiy Map Server

To pass off the Server lab, you will need your netid, your public IP address, and the port you've connected the server to. When you're ready, hover over the "Pass Offs" tab in the menu and select the "Family Map Server" page.

Some Notes:

  • You will either need to run your server from a lab computer, from a computer connected to eduroam wifi, or set up port forwarding on your home router (we will not help you with this option).
  • Ensure that your server is running.
